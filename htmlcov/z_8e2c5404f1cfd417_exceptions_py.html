1"""Public error types for calfkit-tools tool nodes. 

2 

3A tool node signals a *recoverable* failure — one the model can react to (retry, try another 

4source, tell the user) rather than a fault that kills the agent run — by raising ``ModelRetry``. 

5calfkit's tool dispatch recognises this type and surfaces its message to the model as a retry; 

6any other exception escapes to the fault rail and terminates the turn. 

7 

8``ModelRetry`` is re-exported here so node authors depend on a first-party, public name instead 

9of reaching into calfkit's private ``_vendor`` tree. This module is the *single* 

10seam binding calfkit-tools to whatever type calfkit treats as recoverable: when calfkit moves off 

11the vendored pydantic-ai, only this import changes. The contract — that this object is exactly the 

12type calfkit's dispatch catches — is pinned by ``tests/test_exceptions.py`` so a drift fails loudly. 

13""" 

14 

15from __future__ import annotations 

16 

17from calfkit._vendor.pydantic_ai.exceptions import ModelRetry
